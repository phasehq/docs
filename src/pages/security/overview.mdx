# Overview

## Authentication

You'll need to authenticate your requests to access any of the endpoints in the Protocol API. In this guide, we'll look at how authentication works. Protocol offers two ways to authenticate your API requests: Basic authentication and OAuth2 with a token — OAuth2 is the recommended way. {{ className: 'lead' }}


## Data structure
### Phase encrypted is encoded with the following data structure:
```
ph:dataType:ephPublicKey:ciphertext+nonce:tag
```

| Parameter         | Encoding     | Description |
|--------------|-----------|------------|
| **ph** | Plain text      | Helps identify data encrypted with Phase SDKs        |
| **dataType**      | Plain text  | Identifies the type of data encrypted       |
| **ephPublicKey**      | Hex  | An ephemeral ED25519 public key used to derive XChaCha20 shared secret        |
| **ciphertext**      | Hex  | XChaCha20-Poly1305 encrytped data       |
| **nonce**      | Hex  | A randomly generated 192bit initialization vector       |
| **tag**      | Plain text  |        |

Phase app secret data structure:
```
pss:appToken:appID:share0:symmetricWrap
```
| Parameter         | Encoding     | Description |
|--------------|-----------|------------|
| **ph** | Plain text      | Helps identify data encrypted with Phase SDKs        |
| **dataType**      | Plain text  | Identifies the type of data encrypted       |
| **ephPublicKey**      | Hex  | An ephemeral ED25519 public key used for Diffie Hellman handshake        |
| **ciphertext**      | Hex  | XChaCha20-Poly1305 encrytped data       |
| **nonce**      | Hex  | A randomly generated 192bit initialization vector       |
| **tag**      | Plain text  | 5234       |


## Using an SDK

If you use one of our official SDKs, you won't have to worry about any of the above — fetch your access token from the [Protocol dashboard](#) under API settings, and the client library will take care of the rest. All the client libraries use OAuth2 behind the scenes.
